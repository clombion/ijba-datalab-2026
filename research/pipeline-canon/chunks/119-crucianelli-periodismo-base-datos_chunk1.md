## Practical Guide to Data Journalism (21 Mar 2013)

Many people ask me what database journalism is, or data journalism as it is more commonly called. I answer that it is the same investigative journalism as always (including methods from in-depth, precision, analytical, and computer-assisted reporting), to which a few essential components must be added: It works with a large volume of open data, much of which comes from closed formats, meaning that in those cases a prior data-opening task is required. It is team work: alongside a data analyst, a programmer, and an interactive-visualization designer. The inclusion of a developer is vital, especially if you want to design interactive news applications ("News Apps").

### Three Basic Hallmarks of Database Journalism:

1. It offers the audience the **supporting documents** on which the work was based, generally shared from an external platform
2. The reporter **explains their methods**, so that their work withstands critical review; this means that if a reader or another journalist wanted to retrace the same steps using the same documents, they would reach the same conclusion or a different one, should they be able to refute it or find other angles
3. It includes adequate **data visualization**, which will be accompanied by not overly long texts.
