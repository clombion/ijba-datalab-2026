Here is a detailed breakdown of the "School of Data" curriculum, structured over 5 days as requested.

### **Strategic & Pedagogical Introduction**

**The Strategic Approach: The Data Pipeline**
The course is anchored in the **Data Pipeline** framework, a seven-step process (Define, Find, Get, Verify, Clean, Analyze, Present) that structures every data project,. The strategy implied by the content challenges the idea that data journalism is merely about "finding numbers." Instead, it presents data work as a rigorous, investigative cycle:
*   **Iterative Nature:** The process is explicitly described as "non-linear," requiring constant movement back and forth between steps (e.g., cleaning reveals the need for more verification).
*   **Skepticism First:** A significant portion of the curriculum (1.5 days) is dedicated solely to verifying and cleaning data, treating datasets not as absolute truths but as "structured representations of the world" that can be biased, incomplete, or falsified,.
*   **Message Over Form:** In the presentation phase, the strategy prioritizes the journalistic "message" over complex visualization. Data is viewed as a means to tell a story, not an end in itself,.

**The Pedagogical Strategy: "Spreadsheet Thinking"**
The teaching method relies heavily on **active practice** and **accessible tooling**.
*   **Low Barrier to Entry:** Rather than starting with programming languages like Python, the course leverages **Google Sheets** for nearly every stage of the pipeline—from scraping (`IMPORTHTML`) to cleaning (`SPLIT`, `TRIM`) and merging (`INDEX/MATCH`). This allows students to grasp complex data concepts using familiar interfaces,,.
*   **Real-World Application:** Abstract concepts are immediately applied to real datasets, such as the 2022 Legislative election results or local digital inclusion data,.
*   **Group Collaboration:** The use of shared "Jamboards" and collaborative spreadsheets fosters a newsroom-like environment where students solve problems collectively,.

### **Course Outline (5 Days)**

#### **Day 1: Data Acquisition (Récupération)**
*Focus: Identifying data locations, understanding formats, and mastering extraction techniques.*

*   **Morning: The Landscape of Data**
    *   **Introduction to the Data Pipeline:** Overview of the 7 steps.
    *   **Sourcing:** Distinguishing between open data portals, APIs, internal administration systems, and physical archives.
    *   **Formats:** Understanding the hierarchy of machine-readability (CSV/JSON/XLS vs. PDF/Paper).
*   **Afternoon: Extraction Techniques**
    *   **Web Scraping:**
        *   Understanding HTML structure using tools like *X-Ray Goggles*.
        *   **Practice:** Scraping tables from Wikipedia directly into Google Sheets using the `=IMPORTHTML` formula.
        *   Introduction to visual scrapers like **Webscraper.io**.
    *   **APIs:** How to retrieve data via URL parameters (e.g., using the *genderize.io* API).
    *   **PDF Extraction:**
        *   Digital PDFs: Using **Tabula** to unlock tables.
        *   Scanned PDFs: Using OCR tools like **Amazon Textract**.
        *   Document Search: Using **Google Pinpoint** for large document sets.
    *   **AI Assistants:** Using **ChatGPT** to write extraction scripts or troubleshoot formulas.

#### **Day 2: Verification & Structural Cleaning**
*Focus: Assessing data integrity and preparing the file structure.*

*   **Morning: Verification (The 3 Stakes)**
    *   **Concept:** Defining data as a "structured representation of the world" rather than raw truth.
    *   **The 3 Pillars of Verification:**
        *   **Reliability:** Does it represent reality? (Checking for falsified or erroneous data).
        *   **Exhaustiveness:** Is the dataset complete? (Identifying missing values).
        *   **Quality:** Is it structured and documented?.
    *   **Verification Methods:** Statistical checks (min/max/average), cross-referencing with experts, and "common sense" testing.
    *   **Exercise:** Audit the "Digital Inclusion Places" dataset from the Bordeaux Métropole open data portal.
*   **Afternoon: Cleaning Phase 1 – "Toilettage" (Grooming)**
    *   **Goal:** Cleaning the *structure* without altering the *content*.
    *   **Techniques:**
        *   Removing formatting artifacts (e.g., asterisks) using `SUBSTITUTE`.
        *   Splitting combined values (e.g., "5 billion") into value/unit columns using `SPLIT`.
        *   Aligning offset columns caused by bad PDF exports.

#### **Day 3: Advanced Cleaning & Analysis Setup**
*Focus: Content editing, merging datasets, and defining analytical questions.*

*   **Morning: Cleaning Phase 2 – "Edition" & "Rapprochement"**
    *   **Edition (Content Cleaning):**
        *   Handling duplicates using `UNIQUE`.
        *   Categorizing data (e.g., creating age brackets) using `IF` or `IFS`.
        *   Calculating derived metrics (e.g., age from birthdate) using `DATEDIF`.
    *   **Rapprochement (Reconciliation):**
        *   Merging two separate datasets based on a common key (e.g., merging election results with candidate demographics).
        *   **Key Formula:** Mastering `FILTER` and `INDEX/MATCH` for lookups.
*   **Afternoon: Introduction to Analysis**
    *   **Types of Analysis:**
        *   **Descriptive:** What happened? (Mean, Median, Standard Deviation).
        *   **Inferential:** Polling and correlation.
        *   **Predictive:** Machine learning and modeling.
    *   **Methodology:** Breaking a broad theme into specific hypotheses, then converting those into analytical steps.

#### **Day 4: Deep Dive Analysis (Consolidation & Analyse)**
*Focus: Executing descriptive analysis and testing hypotheses.*

*   **Morning: Descriptive Statistics**
    *   **Core Metrics:** Calculation and interpretation of Mean, Median, Standard Deviation, and Maximums.
    *   **Case Study:** Analysis of the 2022 Legislative Election results (2nd tour).
*   **Afternoon: Testing Hypotheses**
    *   **Exercise:** "Which political shades are the most gender-balanced?".
    *   **Workflow:**
        1.  **Formulate Hypothesis:** "Ecological and left-wing parties are closer to parity".
        2.  **Define Analysis Step:** Check if DVG/NUP candidates have 40-60% female representation.
        3.  **Execution:** Using pivot tables or formulas to calculate ratios per party.
        4.  **Refinement:** Reformulating questions based on the findings.

#### **Day 5: Presentation & Cartography**
*Focus: Storytelling, choosing the right visualization, and mapping.*

*   **Morning: The Philosophy of Presentation**
    *   **Presentation vs. Visualization:** Why the message is more important than strict representation,.
    *   **The Triad:** Balancing **Data**, **Form**, and **Message** based on the journalistic angle.
    *   **Selection:** Learning *not* to visualize everything (e.g., only 3 out of 8 articles in a series might need a chart).
*   **Afternoon: Tools & Mapping**
    *   **Visualization Tools:** Overview of **Google Sheets**, **RawGraph**, **Flourish**, and **Datawrapper** for standard charts.
    *   **Cartography:**
        *   **Geographic Data Levels:** Coordinates (Lat/Lon) vs. Geometries (Points, Lines, Polygons).
        *   **File Formats:** Shapefiles, KML, GeoJSON.
        *   **Projections:** Understanding distortions (e.g., Web Mercator).
    *   **Mapping Tools:**
        *   **Khartis:** For statistical maps using administrative boundaries.
        *   **Umap:** For simple interactive points.
        *   **QGIS:** For complex, professional-grade mapping.
